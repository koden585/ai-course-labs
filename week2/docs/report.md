## 3. Ход работы

### 3.1. Архитектура агента

Агент реализован по **семислойной архитектуре** (Interaction → Orchestration → Planning → LLM Core → Memory → Tools → Safety).  
Используется паттерн **ReAct** (Reasoning + Acting).  
Ядро агента находится в `src/agent_core.py`.

### 3.2. Реализованные инструменты

| Инструмент                 | Назначение                          | Параметры             |
|----------------------------|-------------------------------------|-----------------------|
| `search_web`               | Поиск в интернете                   | query, num_results    |
| `calculate`                | Вычисления                          | expression, precision |
| `extract_technical_params` | **Кастомный инструмент** под диплом | drawing_description   |

### 3.3. Кастомный инструмент

**Название:** `extract_technical_params`  
**Назначение:** Распознавание и структурирование технических параметров с графических материалов (чертежей).

**Листинг кода** (`src/tools/custom_tool.py`):
```python
class CustomToolInput(BaseModel):
    drawing_description: str = Field(description="Текстовое описание чертежа")

class CustomTool(BaseTool):
    name = "extract_technical_params"
    description = "Инструмент распознавания технических данных с графических материалов..."
    
    def _run(self, drawing_description: str) -> str:
        # В лабораторной — mock-реализация
        return f"🔧 Распознанные параметры:\n• Деталь: Вал\n• Диаметр: 50 мм\n• Материал: Сталь 45\n• Длина: 200 мм\n• Допуск: h7"
```

**Пример использования**:
```python
response = agent.run("Распознай технические параметры с чертежа: вал диаметром 50 мм из стали 45...")
```
